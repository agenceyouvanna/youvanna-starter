Ne JAMAIS dire "c'est OK" ou "pas grave" quand un problème est identifié dans un retour d'agent ou un process.

**Why:** L'utilisateur veut un pipeline one-shot parfait pour son agence web. Chaque imperfection identifiée doit être corrigée dans le SKILL.md immédiatement.

**How to apply:** À chaque retour d'agent, analyser de manière critique. Si un point pourrait être amélioré, même minime, corriger le SKILL.md tout de suite puis continuer.
